// features

基本機能

ECサイトに備わっている、基本的な機能をご紹介します。
各機能を押下すると、対応する画面に遷移することができます。

🔐

ログイン・アカウント管理

メール＋パスワード認証、ログイン失敗5回でアカウントを30分ロックするセキュリティ機能付き。

👥

会員管理

一般会員は自身の情報を管理。管理者は権限に応じて他会員を閲覧・編集・削除できます。

🔍

商品一覧・検索

カテゴリフィルタと「新着順 / 売れ筋順」の2ソートに対応。注文数を集計したJPQLクエリを実装。

🛍️

買い物かご

商品の追加・個数変更・全削除をセッションで管理。表示時に最新在庫と突き合わせて自動調整。

📦

注文フロー

届け先入力 → 支払方法選択 → 確認 → 完了の4ステップ。確定時に二重の在庫チェックを実施。

📋

注文履歴

一般会員は自分の注文一覧を確認。管理者は全注文を日付降順で閲覧・詳細確認できます。

// Additional Features

追加機能

既存の機能はそのままに、AIを用いた8機能の追加実装を行いました。
Kossumが提供するオリジナルのECサイトです。
各機能を押下すると、対応する画面に遷移することができます。

🔐

セキュリティ管理

基本機能のメール＋パスワード認証に加え、ログイン失敗5回でアカウントを30分ロックするセキュリティ機能を実装。

🌓

ダークモード / ライトモード切替

端末環境や利用シーンに合わせて表示テーマを切り替え可能。視認性や夜間利用の快適さを向上。

⭐

商品のレビュー機能

購入者が星評価とコメントを投稿でき、商品詳細でレビュー一覧を確認。比較検討や信頼感の向上に寄与。

💡

おすすめ機能

閲覧中の商品や購入傾向に応じて関連商品・おすすめ商品を表示。比較検討や追加購入を促し、サイト内の回遊性向上に寄与します。

🔢

買い物かごからの注文数増減

買い物かご画面から商品の注文数を増減できます。数量変更をその場で反映でき、再入力の手間を減らしてスムーズな注文操作に対応。

🚚

配送予定日時指定

注文時に配送予定日や希望時間帯を指定できます。受け取りの利便性向上と配送情報の明確化に対応。

📋

閲覧履歴

一般会員は自分の閲覧履歴を確認することができます。商品の探しやすさを向上。

🖱️

マウスカーソルの変更

サイトにログインすると、マウスカーソルの見た目が切り替わります。

このECサイトの魅力

商品を探す・比較する・購入するまでの流れを、できるだけ迷わず快適に進められるよう設計しました。 レビュー、お気に入り、おすすめ表示、クーポン、配送希望日時指定などを組み合わせることで、 「見つけやすさ」と「買いやすさ」の両方を意識したEC体験を実現しています。

Discover

商品の探しやすさ

カテゴリ検索や並び替えに対応し、目的の商品にたどり着きやすい構成にしています。

Compare

比較しやすい商品選び

レビュー、おすすめ機能によって、ユーザーが複数の商品を比較しやすくしています。

Purchase

スムーズな注文体験

買い物かごでの数量変更、配送希望日時指定により、購入直前の操作性を高めています。

UI / UX

使いやすさへの配慮

ダークモードや画面遷移導線も整え、長時間の閲覧や操作でも負担が少ないUIを意識しました。

// architecture

設計のポイント

標準的なMVC構造に加え、独自の工夫が随所に施されています。

PRG Pattern

二重送信対策

POSTの完了後は必ずリダイレクト（GET）で完了画面を表示し、ブラウザバックによる再送信を防止。

Session管理

入力フォームの保持

入力 → 確認 → 完了の多段フローで、フォーム情報とバリデーションエラーをセッションで引き渡す。

在庫管理

2段階在庫チェック

確認画面表示時と注文確定時の2回チェック。タイムラグによる在庫超過注文を防ぐ設計。

注文履歴

注文時単価の保存

OrderItemに購入時の単価を保存。後から価格が変わっても、注文時の正確な金額を参照可能。

Soft Delete

論理削除

会員・商品・カテゴリはDeleteFlagで論理削除。データは残るため注文履歴との整合性を保てる。

Bean Conversion

BeanToolsサービス

Entity ↔ Form ↔ Bean 間のコピー処理を1クラスに集約。コントローラのコードを簡潔に保つ。
